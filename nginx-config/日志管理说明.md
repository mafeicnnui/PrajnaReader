# Nginx日志管理说明

## Docker环境下的日志管理方案

### 方案对比

| 特性 | 标准输出(/dev/stdout) | 文件日志 |
|------|---------------------|---------|
| 查看方式 | `docker logs` | `tail -f /var/log/nginx/*.log` |
| 持久化 | 否（容器删除即丢失） | 是（需要挂载卷） |
| 按域名分离 | 否（混在一起） | 是（每个域名独立文件） |
| 日志轮转 | Docker自动管理 | 需要手动配置 |
| 磁盘占用 | Docker管理 | 需要监控 |
| 分析工具 | 需要导出 | 直接分析 |
| 推荐场景 | 开发/小规模 | 生产/大规模 |

## 当前配置（推荐）

### 方式1: 标准输出（默认启用）

```nginx
# HTTP服务器
server {
    access_log /dev/stdout;
    error_log /dev/stderr notice;
}

# HTTPS服务器
server {
    access_log /dev/stdout tts_log;
    error_log /dev/stderr warn;
}
```

**查看日志**：
```bash
# 实时查看所有日志
docker logs -f sub-nginx

# 查看最近100行
docker logs --tail=100 sub-nginx

# 查看特定时间段
docker logs --since="2026-03-26T10:00:00" sub-nginx

# 只看错误日志（stderr）
docker logs sub-nginx 2>&1 | grep error
```

**优点**：
- 简单，无需额外配置
- 符合Docker最佳实践
- 日志自动轮转
- 容器删除时自动清理

**缺点**：
- 所有虚拟主机的日志混在一起
- 不能按域名分别查看
- 容器删除后日志丢失

### 方式2: 文件日志（可选）

如果需要持久化日志和按域名分离，可以启用文件日志。

#### 步骤1: 修改nginx配置

编辑 `tts.zhdnpx.cn.conf`，注释掉标准输出，启用文件日志：

```nginx
# HTTPS服务器
server {
    # 注释掉标准输出
    # access_log /dev/stdout tts_log;
    # error_log /dev/stderr warn;
    
    # 启用文件日志
    access_log /var/log/nginx/tts.zhdnpx.cn.access.log tts_log;
    error_log /var/log/nginx/tts.zhdnpx.cn.error.log warn;
}
```

#### 步骤2: 挂载日志目录

修改nginx的docker启动命令，添加日志目录挂载：

```bash
docker run -d \
  --name sub-nginx \
  --restart unless-stopped \
  --network web \
  -p 80:80/tcp \
  -p 443:443/tcp \
  -v /opt/nginx/nginx.conf:/etc/nginx/nginx.conf:ro \
  -v /opt/nginx/conf.d:/etc/nginx/conf.d:ro \
  -v /opt/subscription/www:/usr/share/nginx/html:ro \
  -v /opt/hysteria2/certs:/certs:ro \
  -v /opt/nginx/logs:/var/log/nginx \
  nginx:alpine
```

注意新增的挂载：`-v /opt/nginx/logs:/var/log/nginx`

#### 步骤3: 创建日志目录

```bash
sudo mkdir -p /opt/nginx/logs
sudo chown -R 101:101 /opt/nginx/logs  # nginx用户的UID:GID
sudo chmod 755 /opt/nginx/logs
```

#### 步骤4: 重启nginx

```bash
docker restart sub-nginx
```

#### 步骤5: 查看日志

```bash
# 查看访问日志
tail -f /opt/nginx/logs/tts.zhdnpx.cn.access.log

# 查看错误日志
tail -f /opt/nginx/logs/tts.zhdnpx.cn.error.log

# 查看所有TTS相关日志
tail -f /opt/nginx/logs/tts.zhdnpx.cn.*.log

# 使用grep过滤
tail -f /opt/nginx/logs/tts.zhdnpx.cn.access.log | grep "section/mp3"
```

## 混合方案（推荐用于生产环境）

同时使用两种方式：
- 标准输出：用于快速查看和调试
- 文件日志：用于持久化和分析

```nginx
server {
    # 同时输出到标准输出和文件
    access_log /dev/stdout tts_log;
    access_log /var/log/nginx/tts.zhdnpx.cn.access.log tts_log;
    
    error_log /dev/stderr warn;
    error_log /var/log/nginx/tts.zhdnpx.cn.error.log warn;
}
```

**注意**：这会导致日志写入两次，略微影响性能。

## 日志格式说明

### 标准格式（combined）

```nginx
log_format combined '$remote_addr - $remote_user [$time_local] '
                    '"$request" $status $body_bytes_sent '
                    '"$http_referer" "$http_user_agent"';
```

### TTS自定义格式（包含响应时间）

```nginx
log_format tts_log '$remote_addr - $remote_user [$time_local] '
                   '"$request" $status $body_bytes_sent '
                   '"$http_referer" "$http_user_agent" '
                   'rt=$request_time uct="$upstream_connect_time" '
                   'uht="$upstream_header_time" urt="$upstream_response_time"';
```

**字段说明**：
- `rt`: 请求总时间
- `uct`: 连接上游服务器的时间
- `uht`: 接收上游响应头的时间
- `urt`: 接收上游响应体的时间

**示例日志**：
```
192.168.1.100 - - [26/Mar/2026:10:30:45 +0800] "GET /tts/section/mp3?section_id=ch1-1&voice=male HTTP/2.0" 200 245678 "-" "Mozilla/5.0" rt=2.345 uct="0.001" uht="0.002" urt="2.342"
```

## 日志轮转

### 使用Docker日志轮转（标准输出）

在docker run时配置：

```bash
docker run -d \
  --log-driver json-file \
  --log-opt max-size=10m \
  --log-opt max-file=3 \
  ...
```

或在 `/etc/docker/daemon.json` 中全局配置：

```json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}
```

### 使用logrotate（文件日志）

创建 `/etc/logrotate.d/nginx-tts`：

```
/opt/nginx/logs/tts.zhdnpx.cn.*.log {
    daily
    rotate 7
    missingok
    notifempty
    compress
    delaycompress
    sharedscripts
    postrotate
        docker exec sub-nginx nginx -s reopen
    endscript
}
```

测试配置：
```bash
sudo logrotate -d /etc/logrotate.d/nginx-tts
```

手动执行：
```bash
sudo logrotate -f /etc/logrotate.d/nginx-tts
```

## 日志分析

### 使用GoAccess（实时分析）

```bash
# 安装GoAccess
sudo apt install goaccess

# 实时分析（标准输出）
docker logs sub-nginx | goaccess --log-format=COMBINED -

# 实时分析（文件日志）
goaccess /opt/nginx/logs/tts.zhdnpx.cn.access.log --log-format=COMBINED -o report.html

# 实时监控
tail -f /opt/nginx/logs/tts.zhdnpx.cn.access.log | goaccess --log-format=COMBINED -
```

### 常用分析命令

```bash
# 统计访问最多的IP
awk '{print $1}' /opt/nginx/logs/tts.zhdnpx.cn.access.log | sort | uniq -c | sort -rn | head -10

# 统计访问最多的URL
awk '{print $7}' /opt/nginx/logs/tts.zhdnpx.cn.access.log | sort | uniq -c | sort -rn | head -10

# 统计HTTP状态码
awk '{print $9}' /opt/nginx/logs/tts.zhdnpx.cn.access.log | sort | uniq -c | sort -rn

# 统计响应时间超过1秒的请求
awk '$NF > 1.0 {print $0}' /opt/nginx/logs/tts.zhdnpx.cn.access.log

# 统计每小时的请求数
awk '{print $4}' /opt/nginx/logs/tts.zhdnpx.cn.access.log | cut -d: -f2 | sort | uniq -c

# 查找错误请求（4xx, 5xx）
awk '$9 ~ /^[45]/ {print $0}' /opt/nginx/logs/tts.zhdnpx.cn.access.log
```

## 监控告警

### 使用tail + grep监控错误

```bash
# 监控错误日志
tail -f /opt/nginx/logs/tts.zhdnpx.cn.error.log | while read line; do
    echo "[$(date)] ERROR: $line"
    # 可以在这里添加告警逻辑，如发送邮件或webhook
done
```

### 使用watch监控请求数

```bash
# 每5秒统计一次最近1分钟的请求数
watch -n 5 'tail -n 1000 /opt/nginx/logs/tts.zhdnpx.cn.access.log | wc -l'
```

## 推荐配置

### 开发环境

使用标准输出，简单方便：

```nginx
access_log /dev/stdout tts_log;
error_log /dev/stderr warn;
```

查看日志：
```bash
docker logs -f sub-nginx
```

### 生产环境（小规模）

使用标准输出 + Docker日志轮转：

```bash
docker run -d \
  --log-opt max-size=10m \
  --log-opt max-file=3 \
  ...
```

### 生产环境（大规模）

使用文件日志 + logrotate：

```nginx
access_log /var/log/nginx/tts.zhdnpx.cn.access.log tts_log;
error_log /var/log/nginx/tts.zhdnpx.cn.error.log warn;
```

配置日志轮转和分析工具。

## 故障排查

### 问题1: 看不到日志

检查：
```bash
# 检查nginx配置
docker exec sub-nginx nginx -t

# 检查日志配置
docker exec sub-nginx cat /etc/nginx/conf.d/tts.zhdnpx.cn.conf | grep log

# 检查日志文件权限
docker exec sub-nginx ls -l /var/log/nginx/
```

### 问题2: 日志文件过大

解决：
```bash
# 手动清空日志
sudo truncate -s 0 /opt/nginx/logs/tts.zhdnpx.cn.access.log

# 或删除旧日志
sudo rm /opt/nginx/logs/tts.zhdnpx.cn.access.log.*

# 重新打开日志文件
docker exec sub-nginx nginx -s reopen
```

### 问题3: 日志轮转不工作

检查：
```bash
# 测试logrotate配置
sudo logrotate -d /etc/logrotate.d/nginx-tts

# 查看logrotate日志
sudo cat /var/log/logrotate.log
```

## 总结

**推荐配置**：
- 如果你主要使用 `docker logs` 查看日志，保持当前配置（标准输出）即可
- 如果需要持久化和分析日志，启用文件日志并挂载卷
- 生产环境建议使用文件日志 + logrotate

**当前配置已经是最佳实践**，无需修改，直接使用 `docker logs` 即可。
