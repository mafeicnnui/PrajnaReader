# 音频文件迁移说明

## 变更概述

从按章节生成音频改为按小段（section）生成音频。

## 变更原因

1. **更灵活的播放控制**：支持单段播放、重播、上一段、下一段
2. **更好的用户体验**：用户可以快速跳转到任意段落
3. **减少文件大小**：每个音频文件更小，加载更快
4. **支持连续播放**：可以自动播放整品的所有段落

## 前端代码需要更新的地方

### 1. 音频路径更新

**旧代码**（在 `dizangBook.ts` 中）：
```typescript
{
  id: 'chapter-1',
  title: '第一品 忉利天宫神通品',
  audio: {
    male: '/audio/dizang/chapter-1_male.mp3',
    female: '/audio/dizang/chapter-1_female.mp3',
  }
}
```

**新代码**（需要改为按 section 加载）：
```typescript
// 不再在 chapter 级别定义 audio
// 而是在播放时动态构建 section 的音频路径
const audioPath = `/audio/dizang/sections/${sectionId}_${voice}.mp3`;
```

## 生成新音频文件

```bash
# 1. 导出 sections 数据
cd tts-server
python export_sections.py

# 2. 生成所有音频
python generate_sections.py

# 如果需要代理
python generate_sections.py --proxy http://127.0.0.1:7890
```

## 数据对比

### 旧方案
- 文件数量：6 个（3个章节 × 2种声音）
- 单个文件大小：约 500KB - 2MB（整品）
- 总大小：约 3-6MB

### 新方案
- 文件数量：约 200+ 个（100+ sections × 2种声音）
- 单个文件大小：约 50-200KB（单段）
- 总大小：约 20-40MB
- 优势：更灵活的播放控制，更好的用户体验
