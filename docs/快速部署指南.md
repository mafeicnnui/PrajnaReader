# 般若伴读 - 快速部署指南

本文档提供最简化的部署步骤，适合快速上线。详细说明请参考各专项文档。

## 🎯 部署目标

在 `tts.zhdnpx.cn` 域名下部署：
1. TTS API服务（实时生成音频）
2. App下载页面（介绍和下载APK）

## ⚡ 快速部署（5分钟）

### 前提条件

- 服务器已安装Docker
- 域名 `tts.zhdnpx.cn` 已解析到服务器
- SSL证书已准备好
- Docker网络 `web` 已创建
- Nginx容器 `sub-nginx` 已运行

### 步骤1: 部署TTS服务器（2分钟）

```bash
# 1. 上传代码到服务器
cd /root
git clone <your-repo> PrajnaReader
cd PrajnaReader/tts-server

# 2. 准备数据
mkdir -p cache data
cp data/sections.json data/  # 确保sections.json存在

# 3. 构建并启动
docker build -t prajna-tts-server:latest .

docker run -d \
  --name prajna-tts-server \
  --restart unless-stopped \
  --network web \
  -p 8000:8000 \
  -v $(pwd)/cache:/app/cache \
  -v $(pwd)/data:/app/data:ro \
  prajna-tts-server:latest

# 4. 验证
docker logs prajna-tts-server
curl http://localhost:8000/health
# 预期输出: {"ok":true}
```

### 步骤2: 部署App下载页面（2分钟）

```bash
# 1. 创建目录
sudo mkdir -p /opt/nginx/prajna

# 2. 上传页面文件
cd /root/PrajnaReader
sudo cp tts-server/public/index.html /opt/nginx/prajna/

# 3. 设置权限
sudo chown -R nginx:nginx /opt/nginx/prajna
sudo chmod 755 /opt/nginx/prajna

# 4. 验证文件
ls -l /opt/nginx/prajna/
```

### 步骤3: 配置Nginx（1分钟）

```bash
# 1. 上传配置
cd /root/PrajnaReader
sudo cp nginx-config/tts.zhdnpx.cn.conf /opt/nginx/conf.d/

# 2. 测试配置
docker exec sub-nginx nginx -t

# 3. 重新加载
docker exec sub-nginx nginx -s reload

# 4. 验证
curl https://tts.zhdnpx.cn/health
# 预期输出: {"ok":true}
```

### 步骤4: 发布APK到GitHub

```bash
# 1. 在GitHub项目页面，点击 "Releases"
# 2. 点击 "Create a new release"
# 3. 填写版本号: v1.0.0
# 4. 上传APK文件: Prajna-Reader.apk
# 5. 点击 "Publish release"
```

## 🔧 常见问题

### Q1: TTS服务无法访问

```bash
# 检查容器状态
docker ps | grep prajna-tts-server

# 检查日志
docker logs prajna-tts-server

# 检查网络
docker exec sub-nginx curl http://prajna-tts-server:8000/health
```

### Q2: 页面404

```bash
# 检查文件
docker exec sub-nginx ls -l /usr/share/nginx/prajna/

# 检查nginx配置
docker exec sub-nginx cat /etc/nginx/conf.d/tts.zhdnpx.cn.conf | grep "location = /"

# 如果没有挂载，需要重新创建nginx容器
docker stop sub-nginx
docker rm sub-nginx

docker run -d \
  --name sub-nginx \
  --restart unless-stopped \
  --network web \
  -p 80:80/tcp \
  -p 443:443/tcp \
  -v /opt/nginx/nginx.conf:/etc/nginx/nginx.conf:ro \
  -v /opt/nginx/conf.d:/etc/nginx/conf.d:ro \
  -v /opt/nginx/prajna:/usr/share/nginx/prajna:ro \
  -v /opt/subscription/www:/usr/share/nginx/html:ro \
  -v /opt/hysteria2/certs:/certs:ro \
  nginx:alpine
```

### Q3: 音频播放失败

```bash
# 检查CORS
curl -I -H "Origin: https://example.com" https://tts.zhdnpx.cn/health

# 应该看到:
# Access-Control-Allow-Origin: *

# 测试音频生成
curl -v "https://tts.zhdnpx.cn/tts/section/mp3?section_id=preface-1&voice=male" -o test.mp3

# 播放测试文件
# 如果文件正常，问题在前端；如果文件异常，问题在TTS服务
```
