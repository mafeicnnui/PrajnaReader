## 服务器操作步骤

### 1. 上传新的 sections.json

```bash
# 在本地执行
scp tts-server/data/sections.json root@你的服务器IP:/root/tts-server/data/
```

### 2. 在服务器上生成音频文件

```bash
# SSH 登录服务器
ssh root@你的服务器IP

# 进入 TTS 服务器目录
cd /root/tts-server

# 生成所有音频文件（男声和女声）
python generate_sections.py --voice both

# 或者只生成男声
python generate_sections.py --voice male

# 或者只生成女声
python generate_sections.py --voice female
```

### 3. 等待生成完成

- 总共需要生成 240 个音频文件（120 段 × 2 种声音）
- 预计需要 10-20 分钟
- 脚本会显示进度

### 4. 验证音频文件

```bash
# 检查音频文件数量
ls -l /root/prajna-buddy/public/audio/sutras/dizang/sections/ | wc -l

# 应该显示 240 个文件（或 120 个如果只生成了一种声音）
```

### 5. 测试应用

在手机或浏览器中打开应用，测试第 6 品第 3 段及之后的段落是否能正常播放。

## 如果需要重新导出数据

如果将来修改了地藏经数据，需要重新导出：

```bash
# 在本地项目目录执行
cd prajna-buddy
node export-sections.js

# 然后重复上面的服务器操作步骤
```

## 注意事项

1. 生成音频文件需要网络连接（调用 edge-tts API）
2. 如果服务器在国内，可能需要设置代理：
   ```bash
   export HTTPS_PROXY=http://代理地址:端口
   python generate_sections.py --voice both
   ```
3. 已存在的音频文件会被跳过，不会重复生成
4. 如果需要重新生成某些文件，先删除对应的 mp3 文件再运行脚本
