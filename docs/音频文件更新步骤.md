## 服务器操作步骤（Docker 部署）

### 1. 上传新的 sections.json

```bash
# 在本地执行
scp tts-server/data/sections.json root@你的服务器IP:/root/tts-server/data/
```

### 2. 在 Docker 容器内生成音频文件

由于 TTS 服务器运行在 Docker 容器中，data 和 cache 目录都是通过 volume 挂载的，所以**不需要重新构建镜像**。

```bash
# SSH 登录服务器
ssh root@你的服务器IP

# 进入 Docker 容器
docker exec -it prajna-tts-server bash

# 在容器内生成所有音频文件（男声和女声）
python generate_sections.py --voice both

# 或者只生成男声
python generate_sections.py --voice male

# 或者只生成女声
python generate_sections.py --voice female

# 生成完成后退出容器
exit
```

**或者直接在宿主机执行（推荐）：**

```bash
# 直接在宿主机执行容器内的命令
docker exec prajna-tts-server python generate_sections.py --voice both
```

### 3. 等待生成完成

- 总共需要生成 240 个音频文件（120 段 × 2 种声音）
- 预计需要 10-20 分钟
- 脚本会显示进度

### 4. 验证音频文件

```bash
# 检查容器内 cache 目录的音频文件数量
docker exec prajna-tts-server ls -l /app/cache | wc -l

# 或者检查宿主机挂载的 cache 目录
ls -l /root/tts-server/cache | wc -l

# 应该显示 240+ 个文件（120 段 × 2 种声音）
```

### 5. 测试应用

在手机或浏览器中打开应用，测试第 6 品第 3 段及之后的段落是否能正常播放。

## 如果需要重新导出数据

如果将来修改了地藏经数据，需要重新导出：

```bash
# 在本地项目目录执行
cd prajna-buddy
node export-sections.js

# 然后重复上面的服务器操作步骤
```

## 注意事项

1. **不需要重新构建 Docker 镜像**：data 和 cache 目录都是通过 volume 挂载的
2. 生成音频文件需要网络连接（调用 edge-tts API）
3. 如果服务器在国内，可能需要设置代理（在 docker-compose.yml 中配置）
4. 已存在的音频文件会被跳过，不会重复生成
5. 如果需要重新生成某些文件，先删除对应的 mp3 文件再运行脚本
6. 音频文件存储在 `tts-server/cache` 目录（宿主机）或 `/app/cache`（容器内）

```bash
# 重启容器（如果需要）

docker exec -it prajna-tts-server bash

# 查看容器内的数据文件
docker exec prajna-tts-server cat /app/data/sections.json | head -20

# 查看容器内的音频文件
docker exec prajna-tts-server ls -lh /app/cache
```
