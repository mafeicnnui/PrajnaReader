# 地藏经音频生成说明

## 概述

为了支持更灵活的音频播放功能，我们将音频生成从"按品"改为"按小段（section）"。这样可以：

1. 支持整品连续播放
2. 支持单段播放和重播
3. 减少单个音频文件的大小
4. 提供更好的用户体验

## ⚠️ 重要变更

**旧的按章节生成的音频文件已被删除**，包括：
- `chapter-1_male.mp3` / `chapter-1_female.mp3`
- `kaijingji_male.mp3` / `kaijingji_female.mp3`
- `huixiangji_male.mp3` / `huixiangji_female.mp3`

新的音频文件将按小段生成，存放在 `sections/` 子目录中。

## 音频生成步骤

### 1. 安装依赖

```bash
cd tts-server
pip install -r requirements.txt
```

### 2. 导出 sections 数据

从 TypeScript 源文件导出所有小段数据：

```bash
python export_sections.py
```

这会生成 `tts-server/data/sections.json` 文件。

### 3. 生成音频文件

生成所有小段的音频（男声和女声）：

```bash
python generate_sections.py
```

如果网络受限，可以使用代理：

```bash
python generate_sections.py --proxy http://127.0.0.1:7890
```

只生成特定小段：

```bash
python generate_sections.py --sections ch1-1 ch1-2 ch1-3
```

只生成男声或女声：

```bash
python generate_sections.py --voice male
# 或
python generate_sections.py --voice female
```

## 音频文件结构

生成的音频文件位于：`prajna-buddy/public/audio/dizang/sections/`

文件命名规则：`{sectionId}_{voice}.mp3`

示例：
```
sections/
├── preface-1_male.mp3      # 香赞-男声
├── preface-1_female.mp3    # 香赞-女声
├── preface-2_male.mp3      # 觉林菩萨偈-男声
├── preface-2_female.mp3    # 觉林菩萨偈-女声
├── ch1-1_male.mp3          # 第一品第1段-男声
├── ch1-1_female.mp3        # 第一品第1段-女声
├── ch1-2_male.mp3          # 第一品第2段-男声
├── ch1-2_female.mp3        # 第一品第2段-女声
...
```

## 前端播放功能

### 当前实现

前端已经支持章节级别的音频播放，位于 `SutraChapterReader.tsx`。

### 需要的改进

为了支持小段级别的播放，需要：

1. **音频路径更新**：
   - 从 `/audio/dizang/{chapterId}_{voice}.mp3`
   - 改为 `/audio/dizang/sections/{sectionId}_{voice}.mp3`

2. **播放控制增强**：
   - 添加"上一段"、"下一段"按钮
   - 支持自动连续播放整品
   - 显示当前播放的段落
   - 支持单段循环播放

3. **UI 改进**：
   - 在每个段落旁边添加播放按钮
   - 高亮显示当前播放的段落
   - 显示播放进度

## 注意事项

1. **网络要求**：生成音频需要连接到 Microsoft Edge TTS 服务
2. **代理设置**：如果网络受限，需要设置代理
3. **生成时间**：完整生成所有音频大约需要 10-20 分钟
4. **存储空间**：每个音频文件约 50-200KB，总计约 20-40MB
5. **增量生成**：已存在的文件会被跳过，支持增量生成

## 故障排除

### 问题：WebSocket handshake failed

**原因**：网络限制，无法连接到 Microsoft Edge TTS 服务

**解决方案**：
```bash
# 设置代理
python generate_sections.py --proxy http://127.0.0.1:7890

# 或设置环境变量
export HTTPS_PROXY=http://127.0.0.1:7890
python generate_sections.py
```

### 问题：找不到 sections.json

**原因**：未运行导出脚本

**解决方案**：
```bash
python export_sections.py
```

### 问题：某些 section 生成失败

**原因**：网络不稳定或文本格式问题

**解决方案**：
```bash
# 重新生成失败的 sections
python generate_sections.py --sections ch1-5 ch1-6
```
