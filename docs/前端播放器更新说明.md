# 前端播放器更新说明

## 更新概述

前端播放器已更新，支持三种音频加载方式：
1. TTS API实时生成（推荐用于开发和新增内容）
2. 远程静态文件服务器（推荐用于生产环境）
3. 本地静态文件（用于离线开发）

## 主要变更

### 1. 音频URL生成逻辑

更新了 `getSectionAudioUrl` 函数，支持多种音频源：

```typescript
const getSectionAudioUrl = (sectionId: string, voiceType: VoiceId): string => {
  // 优先级：TTS API > 远程静态文件 > 本地静态文件
  const ttsApiUrl = import.meta.env.VITE_TTS_API_URL;
  const audioBaseUrl = import.meta.env.VITE_AUDIO_BASE_URL || '';
  
  if (ttsApiUrl) {
    // 使用TTS API实时生成
    return `${ttsApiUrl}/tts/section/mp3?section_id=${sectionId}&voice=${voiceType}`;
  }
  
  // 使用静态文件
  return `${audioBaseUrl}/audio/sutras/dizang/sections/${sectionId}_${voiceType}.mp3`;
};
```

### 2. 音频目录结构调整

从：
```
public/audio/dizang/sections/
```

改为：
```
public/audio/sutras/dizang/sections/
```

这样可以支持未来扩展其他经典（如金刚经、心经等）。

### 3. 环境变量配置

新增两个环境变量：

- `VITE_TTS_API_URL`: TTS API服务器地址
- `VITE_AUDIO_BASE_URL`: 静态文件服务器地址

## 使用方式

### 开发环境

#### 方式1: 使用本地TTS服务（推荐）

1. 启动TTS服务：
```bash
cd tts-server
uvicorn main:app --reload
```

2. 配置环境变量（`.env.development`）：
```env
VITE_TTS_API_URL=http://localhost:8000
```

3. 启动前端：
```bash
cd prajna-buddy
npm run dev
```

优点：
- 无需预先生成音频文件
- 可以实时测试新内容
- 自动缓存，提高效率

#### 方式2: 使用本地静态文件

1. 生成音频文件：
```bash
cd tts-server
python generate_sections.py
```

2. 配置环境变量（`.env.development`）：
```env
VITE_TTS_API_URL=
VITE_AUDIO_BASE_URL=
```

3. 启动前端：
```bash
cd prajna-buddy
npm run dev
```

### 生产环境

#### 方案1: TTS API服务

适用于：
- 内容频繁更新
- 用户量较小
- 存储空间有限

配置（`.env.production`）：
```env
VITE_TTS_API_URL=https://tts.zhdnpx.cn
```

部署TTS服务：
```bash
cd tts-server
docker-compose up -d
```

配置nginx反向代理（参考 [TTS服务器部署说明](./TTS服务器部署说明.md)）

#### 方案2: 静态文件服务器

适用于：
- 内容相对稳定
- 用户量较大
- 需要CDN加速

配置（`.env.production`）：
```env
VITE_TTS_API_URL=
VITE_AUDIO_BASE_URL=https://files.zhdnpx.cn
```

部署步骤：
1. 生成音频文件
2. 上传到服务器
3. 配置nginx（参考 [音频文件服务器部署说明](./音频文件服务器部署说明.md)）

#### 方案3: 混合模式（推荐）

结合两种方案的优点：

配置（`.env.production`）：
```env
VITE_TTS_API_URL=https://tts.zhdnpx.cn
VITE_AUDIO_BASE_URL=https://files.zhdnpx.cn
```

工作流程：
1. 优先使用TTS API生成音频
2. 如果TTS API失败，fallback到静态文件
3. 常用内容预先生成并上传到静态服务器
4. 新增内容通过TTS API实时生成

## 播放功能

### 1. 单段播放

点击每个段落旁边的播放按钮，播放该段音频。

### 2. 连续播放

开启"连续播放"开关后，播放完当前段落会自动播放下一段。

### 3. 单段循环

开启"单段循环"按钮后，当前段落会循环播放。

注意：单段循环和连续播放互斥，开启一个会自动关闭另一个。

### 4. 播放控制

迷你播放器提供以下控制：
- 播放/暂停
- 上一段
- 下一段
- 单段循环
- 进度条拖动
- 播放速度调整（未来功能）

### 5. 音色选择

支持两种音色：
- 男声（zh-CN-YunxiNeural）
- 女声（zh-CN-XiaoxiaoNeural）

切换音色后，会自动加载对应音色的音频。

## 性能优化

### 1. 音频缓存

- TTS API会自动缓存生成的音频
- 浏览器会缓存已下载的音频文件
- 静态文件服务器配置了长期缓存（30天）

### 2. Range请求支持

所有音频URL都支持HTTP Range请求，实现：
- 流式播放（无需等待完整下载）
- 进度条拖动
- 断点续传

### 3. 错误处理

- 音频加载失败时显示错误提示
- 自动重试机制（未来功能）
- Fallback到备用音频源（混合模式）
