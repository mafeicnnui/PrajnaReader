# 音频文件快速更新指南（Docker 版）

## 最简单的方法（推荐）

TTS 服务器会**自动生成**音频文件，你只需要：

### 1. 上传新的 sections.json

```bash
scp tts-server/data/sections.json root@你的服务器:/root/tts-server/data/
```

### 2. 重启 Docker 容器

```bash
ssh root@你的服务器
docker restart prajna-tts-server
```

### 3. 完成！

前端访问时，TTS 服务器会自动生成音频文件并缓存。首次访问会慢一点（生成音频），之后就快了。

---

## 预生成所有音频（可选）

如果你想提前生成所有音频文件，避免用户首次访问时等待：

```bash
# SSH 登录服务器
ssh root@你的服务器

# 在 Docker 容器内生成所有音频（推荐）
docker exec prajna-tts-server python generate_sections.py --voice both

# 或者只生成男声
docker exec prajna-tts-server python generate_sections.py --voice male
```

生成过程需要 10-20 分钟，会生成 240 个音频文件（120 段 × 2 种声音）。

---

## 验证

```bash
# 查看已生成的音频文件数量
docker exec prajna-tts-server ls /app/cache | wc -l

# 或者在宿主机查看
ls /root/tts-server/cache | wc -l
```

## 常见问题

### Q: 需要重新构建 Docker 镜像吗？
A: **不需要**！data 和 cache 目录都是通过 volume 挂载的，只需要更新文件和重启容器。

### Q: 如果生成失败怎么办？
A: 检查容器日志：
```bash
docker logs -f prajna-tts-server
```

可能的原因：
- 网络问题（edge-tts 需要访问微软服务器）
- 需要设置代理（在 docker-compose.yml 中配置）

### Q: 如何清空缓存重新生成？
A: 删除 cache 目录的文件：
```bash
# 删除所有缓存
rm -rf /root/tts-server/cache/*

# 或者只删除 section 相关的
rm -f /root/tts-server/cache/section_*

# 然后重新生成
docker exec prajna-tts-server python generate_sections.py --voice both
```

### Q: 音频文件存储在哪里？
A: 
- 容器内：`/app/cache/`
- 宿主机：`/root/tts-server/cache/`（通过 volume 挂载）

### Q: 文件名格式是什么？
A: `section_{section_id}_{voice}_{hash}.mp3`
- 例如：`section_ch1-1_male_a1b2c3d4.mp3`
- hash 是根据文本内容生成的，确保相同文本使用相同缓存
