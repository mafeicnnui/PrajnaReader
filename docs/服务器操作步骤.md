# 修复nginx 404错误 - 服务器操作步骤

## 问题原因

服务器上的nginx配置文件缺少根路径（`location = /`）的配置，导致访问 `https://tts.zhdnpx.cn/` 时返回404。

## 解决方案

### 方式1: 从本地上传配置文件（推荐）

在本地执行：

```bash
# 上传更新后的配置文件
scp nginx-config/tts.zhdnpx.cn.conf root@your-server:/opt/nginx/conf.d/

# SSH到服务器
ssh root@your-server

# 测试配置
docker exec sub-nginx nginx -t

# 重新加载
docker exec sub-nginx nginx -s reload

# 验证
curl -I https://tts.zhdnpx.cn/
```

### 方式2: 在服务器上直接编辑

在服务器上执行：

```bash
# 备份当前配置
cp /opt/nginx/conf.d/tts.zhdnpx.cn.conf /opt/nginx/conf.d/tts.zhdnpx.cn.conf.backup

# 编辑配置文件
nano /opt/nginx/conf.d/tts.zhdnpx.cn.conf
```

在 `add_header 'Access-Control-Max-Age' '1728000' always;` 这行之后，添加以下内容：

```nginx
    # 根路径 - App介绍和下载页面
    location = / {
        alias /usr/share/nginx/prajna/index.html;
    }

    # 静态资源（如果有）
    location /static/ {
        alias /usr/share/nginx/prajna/static/;
        expires 7d;
        add_header Cache-Control "public";
    }
```

保存后执行：

```bash
# 测试配置
docker exec sub-nginx nginx -t

# 重新加载
docker exec sub-nginx nginx -s reload

# 验证
curl -I https://tts.zhdnpx.cn/
```

### 方式3: 使用cat命令直接写入（最快）

在服务器上执行以下命令，找到正确的插入位置：

```bash
# 1. 查看当前配置中CORS配置的位置
docker exec sub-nginx cat /etc/nginx/conf.d/tts.zhdnpx.cn.conf | grep -n "Access-Control-Max-Age"

# 2. 备份配置
cp /opt/nginx/conf.d/tts.zhdnpx.cn.conf /opt/nginx/conf.d/tts.zhdnpx.cn.conf.backup

# 3. 在服务器上创建临时文件
cat > /tmp/root-location.conf << 'EOF'

    # 根路径 - App介绍和下载页面
    location = / {
        alias /usr/share/nginx/prajna/index.html;
    }

    # 静态资源（如果有）
    location /static/ {
        alias /usr/share/nginx/prajna/static/;
        expires 7d;
        add_header Cache-Control "public";
    }
EOF

# 4. 手动编辑配置文件，将上述内容插入到合适位置
nano /opt/nginx/conf.d/tts.zhdnpx.cn.conf
```

## 完整的配置文件参考

如果需要完整重写配置文件，可以参考项目中的 `nginx-config/tts.zhdnpx.cn.conf` 文件。

关键部分应该包含：

```nginx
server {
    listen 443 ssl;
    listen [::]:443 ssl;
    http2 on;
    server_name tts.zhdnpx.cn;

    # ... SSL配置 ...

    # CORS配置
    add_header 'Access-Control-Allow-Origin' '*' always;
    add_header 'Access-Control-Allow-Methods' 'GET, POST, HEAD, OPTIONS' always;
    add_header 'Access-Control-Allow-Headers' 'Origin, X-Requested-With, Content-Type, Accept, Range, Authorization' always;
    add_header 'Access-Control-Expose-Headers' 'Content-Length, Content-Range, Content-Type' always;
    add_header 'Access-Control-Max-Age' '1728000' always;

    # 根路径 - App介绍和下载页面
    location = / {
        alias /usr/share/nginx/prajna/index.html;
    }

    # 静态资源
    location /static/ {
        alias /usr/share/nginx/prajna/static/;
        expires 7d;
        add_header Cache-Control "public";
    }

    # 健康检查
    location = /health {
        proxy_pass http://tts_backend;
        # ...
    }

    # TTS API
    location /tts/ {
        proxy_pass http://tts_backend;
        # ...
    }

    # ... 其他location配置 ...
}
```

## 故障排查

如果仍然404：

```bash
# 1. 确认文件存在
docker exec sub-nginx ls -la /usr/share/nginx/prajna/
# 应该看到 index.html

# 2. 确认文件可读
docker exec sub-nginx cat /usr/share/nginx/prajna/index.html | head -10
# 应该看到HTML内容

# 3. 检查nginx错误日志
docker logs sub-nginx 2>&1 | tail -50

# 4. 检查nginx访问日志
docker logs sub-nginx 2>&1 | grep "GET / "

# 5. 测试从容器内访问
docker exec sub-nginx curl -I http://localhost/
```

## 注意事项

1. 使用 `alias` 而不是 `root`，因为 `location = /` 需要精确匹配
2. 确保文件路径正确：`/usr/share/nginx/prajna/index.html`
3. 确保nginx容器已挂载该目录：`-v /opt/nginx/prajna:/usr/share/nginx/prajna:ro`
4. 修改配置后必须重新加载：`docker exec sub-nginx nginx -s reload`
