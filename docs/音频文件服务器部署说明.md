# 音频文件服务器部署说明

## 概述

本文档说明如何在云主机上部署音频文件服务器，用于托管地藏经及其他经典的音频文件。

## 目录结构设计

### 新的音频目录结构

为了支持多经典扩展，音频文件目录结构已重新设计：

```
prajna-buddy/public/audio/
└── sutras/                    # 经典音频根目录
    └── dizang/                # 地藏经
        └── sections/          # 小段音频
            ├── preface-1_male.mp3
            ├── preface-1_female.mp3
            ├── ch1-1_male.mp3
            ├── ch1-1_female.mp3
            └── ...
```

未来可以添加其他经典：
```
audio/sutras/
├── dizang/                    # 地藏经
├── jingang/                   # 金刚经
├── xinjing/                   # 心经
└── ...
```

### 服务器端目录结构

在nginx服务器上，建议使用相同的目录结构：

```
/usr/share/nginx/files/
└── audio/
    └── sutras/
        └── dizang/
            └── sections/
                ├── preface-1_male.mp3
                ├── preface-1_female.mp3
                └── ...
```

## Nginx配置

### 1. 配置文件位置

配置文件已生成在项目根目录：`nginx-config/files.zhdnpx.cn.conf`

### 2. 部署步骤

#### 步骤1: 复制配置文件到服务器

```bash
# 将配置文件复制到服务器的nginx配置目录
scp nginx-config/files.zhdnpx.cn.conf user@your-server:/opt/nginx/conf.d/
```

#### 步骤2: 创建文件存储目录

```bash
# SSH登录到服务器
ssh user@your-server

# 创建文件存储目录
sudo mkdir -p /opt/nginx/files/audio/sutras/dizang/sections

# 设置权限
sudo chown -R nginx:nginx /opt/nginx/files
sudo chmod -R 755 /opt/nginx/files
```

#### 步骤3: 上传音频文件

```bash
# 从本地上传音频文件到服务器
# 方式1: 使用scp
scp -r prajna-buddy/public/audio/sutras/dizang/sections/*.mp3 \
    user@your-server:/opt/nginx/files/audio/sutras/dizang/sections/

# 方式2: 使用rsync（推荐，支持断点续传）
rsync -avz --progress \
    prajna-buddy/public/audio/sutras/dizang/sections/ \
    user@your-server:/opt/nginx/files/audio/sutras/dizang/sections/
```

#### 步骤4: 配置SSL证书

确保SSL证书文件存在：
- `/opt/hysteria2/certs/files.zhdnpx.cn.crt`
- `/opt/hysteria2/certs/files.zhdnpx.cn.key`

如果证书不存在，需要申请SSL证书（可以使用Let's Encrypt）。

#### 步骤5: 更新Docker挂载配置

修改nginx的docker启动命令，添加文件目录挂载：

```bash
docker stop sub-nginx
docker rm sub-nginx

docker run -d --name sub-nginx --restart unless-stopped --network web \
  -p 80:80/tcp -p 443:443/tcp \
  -v /opt/nginx/nginx.conf:/etc/nginx/nginx.conf:ro \
  -v /opt/nginx/conf.d:/etc/nginx/conf.d:ro \
  -v /opt/subscription/www:/usr/share/nginx/html:ro \
  -v /opt/nginx/files:/usr/share/nginx/files:ro \
  -v /opt/hysteria2/certs:/certs:ro \
  nginx:alpine
```

注意新增的挂载：`-v /opt/nginx/files:/usr/share/nginx/files:ro`

#### 步骤6: 重启nginx

```bash
# 重启nginx容器
docker restart sub-nginx

# 查看日志确认启动成功
docker logs sub-nginx
```

#### 步骤7: 测试访问

```bash
# 测试HTTP重定向
curl -I http://files.zhdnpx.cn

# 测试HTTPS访问
curl -I https://files.zhdnpx.cn/audio/sutras/dizang/sections/ch1-1_male.mp3

# 测试CORS
curl -I -H "Origin: https://your-app-domain.com" \
    https://files.zhdnpx.cn/audio/sutras/dizang/sections/ch1-1_male.mp3
```

## 前端配置

### 环境变量配置

在前端项目中，可以通过环境变量配置音频服务器地址。

#### 开发环境（使用本地音频）

创建 `prajna-buddy/.env.development`：

```env
# 留空表示使用本地音频文件
VITE_AUDIO_BASE_URL=
```

#### 生产环境（使用远程服务器）

创建 `prajna-buddy/.env.production`：

```env
# 使用远程音频服务器
VITE_AUDIO_BASE_URL=https://files.zhdnpx.cn
```

### 代码说明

前端代码已更新，`getSectionAudioUrl` 函数会自动根据环境变量选择音频源：

```typescript
const getSectionAudioUrl = (sectionId: string, voiceType: VoiceId): string => {
  const audioBaseUrl = import.meta.env.VITE_AUDIO_BASE_URL || '';
  return `${audioBaseUrl}/audio/sutras/dizang/sections/${sectionId}_${voiceType}.mp3`;
};
```

## 安全建议

1. 定期更新SSL证书
2. 监控访问日志，防止恶意下载
3. 考虑添加访问频率限制（rate limiting）
4. 定期备份音频文件

## 故障排查

### 问题1: 音频无法播放

检查：
- nginx配置是否正确
- 文件权限是否正确（755）
- CORS配置是否生效
- SSL证书是否有效

### 问题2: 跨域错误

确认nginx配置中的CORS头部已正确设置，特别是OPTIONS请求的处理。

### 问题3: 音频加载慢

考虑：
- 启用CDN加速
- 优化音频文件大小
- 检查服务器带宽
