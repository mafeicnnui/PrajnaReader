# 音频系统架构说明

## 系统概述

地藏经音频系统支持三种部署模式：
1. TTS API实时生成模式
2. 静态文件服务器模式
3. 混合模式（推荐）

## 数据流程

### 1. TTS API模式

```
用户点击播放
    ↓
前端请求: GET /tts/section/mp3?section_id=ch1-1&voice=male
    ↓
TTS服务器检查缓存
    ↓
    ├─ 缓存存在 → 直接返回mp3文件
    └─ 缓存不存在 → 调用edge-tts生成 → 保存缓存 → 返回mp3文件
    ↓
浏览器播放音频
```

### 2. 静态文件模式

```
用户点击播放
    ↓
前端请求: GET /audio/sutras/dizang/sections/ch1-1_male.mp3
    ↓
Nginx返回静态文件
    ↓
浏览器播放音频
```

### 3. 混合模式

```
用户点击播放
    ↓
前端首先尝试TTS API
    ↓
    ├─ 成功 → 播放音频
    └─ 失败 → Fallback到静态文件服务器
        ↓
        ├─ 成功 → 播放音频
        └─ 失败 → 显示错误提示
```

## 部署方案对比

| 特性 | TTS API | 静态文件 | 混合模式 |
|------|---------|----------|----------|
| 响应速度 | 慢（首次1-3秒） | 快（<100ms） | 快 |
| 存储需求 | 低（仅缓存） | 高（全部文件） | 中 |
| 服务器负载 | 高 | 低 | 中 |
| 扩展性 | 优秀 | 一般 | 优秀 |
| 维护成本 | 高 | 低 | 中 |
| 适用场景 | 开发/测试 | 生产环境 | 生产环境 |
| 推荐指数 | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

## 环境配置

### 开发环境

```env
# .env.development
VITE_TTS_API_URL=http://localhost:8000
VITE_AUDIO_BASE_URL=
```

启动服务：
```bash
# 终端1: 启动TTS服务
cd tts-server
uvicorn main:app --reload

# 终端2: 启动前端
cd prajna-buddy
npm run dev
```

### 生产环境 - TTS API模式

```env
# .env.production
VITE_TTS_API_URL=https://tts.zhdnpx.cn
VITE_AUDIO_BASE_URL=
```

部署：
```bash
# 部署TTS服务
cd tts-server
docker-compose up -d

# 配置nginx反向代理
# 域名: tts.zhdnpx.cn -> http://localhost:8000
```

### 生产环境 - 静态文件模式

```env
# .env.production
VITE_TTS_API_URL=
VITE_AUDIO_BASE_URL=https://files.zhdnpx.cn
```

部署：
```bash
# 1. 生成音频文件
cd tts-server
python generate_sections.py

# 2. 上传到服务器
rsync -avz prajna-buddy/public/audio/sutras/ \
    user@server:/opt/nginx/files/audio/sutras/

# 3. 配置nginx静态文件服务
# 使用 nginx-config/files.zhdnpx.cn.conf
```

### 生产环境 - 混合模式（推荐）

```env
# .env.production
VITE_TTS_API_URL=https://tts.zhdnpx.cn
VITE_AUDIO_BASE_URL=https://files.zhdnpx.cn
```

部署：
```bash
# 1. 部署TTS服务
cd tts-server
docker-compose up -d

# 2. 生成并上传常用音频
python generate_sections.py
rsync -avz prajna-buddy/public/audio/sutras/ \
    user@server:/opt/nginx/files/audio/sutras/

# 3. 配置两个nginx服务
# - tts.zhdnpx.cn (TTS API反向代理)
# - files.zhdnpx.cn (静态文件服务)
```

## API接口

### TTS服务器API

#### 1. 健康检查
```
GET /health
```

#### 2. 列出所有sections
```
GET /sections
```

#### 3. 获取section详情
```
GET /section/{section_id}
```

#### 4. 生成section音频（推荐）
```
GET /tts/section/mp3?section_id={id}&voice={voice}&force={bool}
```

参数：
- `section_id`: section的ID（必填）
- `voice`: male 或 female（默认：male）
- `force`: 是否强制重新生成（默认：false）

#### 5. 流式生成音频
```
GET /tts/section/stream?section_id={id}&voice={voice}
```

#### 6. 任意文本转语音
```
GET /tts/text/mp3?text={text}&voice={voice}&force={bool}
```

### 静态文件服务器

```
GET /audio/sutras/dizang/sections/{section_id}_{voice}.mp3
```

示例：
```
GET /audio/sutras/dizang/sections/ch1-1_male.mp3
GET /audio/sutras/dizang/sections/preface-1_female.mp3
```

## 性能优化

### 1. TTS服务优化

- 使用多worker提高并发：`--workers 4`
- 配置合理的缓存策略
- 使用代理加速edge-tts访问
- 定期清理旧缓存

### 2. 静态文件优化

- 启用gzip压缩
- 配置长期缓存（30天）
- 使用CDN加速
- 启用HTTP/2

- 音频预加载（未来功能）

## 故障排查

### 常见问题

1. **音频无法播放**
   - 检查网络连接
   - 检查CORS配置
   - 查看浏览器控制台错误

2. **TTS生成失败**
   - 检查edge-tts服务可用性
   - 检查代理配置
   - 查看服务器日志

3. **响应速度慢**
   - 增加TTS worker数量
   - 使用静态文件模式
   - 启用CDN加速

4. **缓存占用过大**
   - 定期清理旧缓存
   - 调整缓存策略
   - 使用静态文件模式
